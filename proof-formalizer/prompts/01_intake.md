# Stage 1: Intake

读取用户输入的证明，产出 `intake.md`。

## 输入

用户提供的证明（LaTeX / Markdown / 自然语言 / 混合格式）。

## 处理步骤

1. **提取定理陈述**：找到"定理/命题/引理"声明。没有显式声明的从证明反推结论。
2. **分离证明主体**：保留原文不改写。
3. **识别符号**：列所有数学符号及其 Lean 对应，标注隐含类型（n→Nat, x→Real）。
4. **识别前置概念**：证明引用但未定义的概念，列出 Mathlib 可能位置。
5. **标记输入格式**：LaTeX / Markdown / 纯文本 / 混合。

## 输出

写入 `intake.md`：

```markdown
# Intake: <问题标识>

## 定理陈述
**原文**：<原始文本>
**Lean 猜测**：
```lean
theorem <name> : <statement> := by
```

## 证明主体
<原样保留>

## 符号表
| 原文 | Lean 对应 | 隐含类型 |
|-----|----------|---------|

## 前置概念
| 概念 | Mathlib 位置 | 需确认 |
|-----|-------------|-------|

## 输入格式
<LaTeX / Markdown / 纯文本 / 混合>
```

## 注意

- 不改写证明
- 定理陈述不精确时在 Lean 猜测旁标注"需 Segment 确认"
- 外部定理引用（"由 Cauchy-Schwarz"）列出但不展开
